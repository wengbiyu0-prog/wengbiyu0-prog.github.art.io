# EDIMAGE WORLD 知识库

**版本**：v0.1  
**用途**：储存主理人和用户上传的创意文本。  
**当前状态**：MVP 本地知识库。

> 知识库是 EDIMAGE WORLD 的记忆系统。  
> 每一段存入的文字，都可能在未来长成一处地点、一个物件、一句对白，或者一条全新小径。

---

## 一、核心原则

1. **第一优先资源**  
   文本生成时，知识库召回内容优先于任何其他来源（用户当前 idea、卡片、路径历史、智能体推理）。当知识库为空时，生成完全依赖智能体推理。

2. **个性化显影**  
   知识库越丰富，EDIMAGE WORLD 的文本越应体现创作者长期积累的意象、语言、地点和叙事偏好。上传不是为了堆砌，而是为了长出独特地貌。

3. **权限诚实**  
   每一条文本必须明确权限状态。系统严格遵守用户对“个人生成”“公共生成”“公开展示”的选择。

4. **保留来源**  
   所有入库条目需保留作者、上传时间、原始类型，避免未来产生归属与使用争议。

5. **可被拆分与关联**  
   知识库中的文本不是静态档案。系统可提取其中的意象、标签、风格，并将其作为未来生成的世界节点素材。

## 二、知识库字段定义

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 标题 | 字符串 | 是 | 条目标题，可由用户填写或系统自动提取 |
| 正文 | 文本 | 是 | 原始内容，保持完整 |
| 作者 | 字符串 | 是 | 上传者标识（用户ID 或 “主理人”） |
| 来源 | 字符串 | 否 | 如“个人旧稿”“梦境记录于2026-05-01” |
| 创建时间 | ISO 时间戳 | 是 | 首次入库时间 |
| 类型标签 | 数组 | 是 | 对应 PRD 中的类型标签体系，2～4 个 |
| 文风标签 | 数组 | 是 | 对应 PRD 中的文风标签，1～2 个 |
| 意象关键词 | 数组 | 是 | 提取出的核心意象（便利店、雨、旧钥匙等） |
| 权限状态 | 枚举 | 是 | 见下方权限说明 |
| 是否允许个人生成引用 | 布尔 | 是 | 是否可在上传者自己的生成中使用 |
| 是否允许公共生成引用 | 布尔 | 是 | 是否可进入公共生成池 |
| 是否公开展示 | 布尔 | 是 | 是否可在公共知识库中被他人阅读 |
| 文本类型 | 枚举 | 是 | 见下方文本类型 |

## 四、权限状态

| 状态 | 含义 | 个人生成引用 | 公共生成引用 | 公开展示 |
| --- | --- | --- | --- | --- |
| **私密** | 仅作者可见，仅作者可用于生成 | ✅ 是 | ❌ 否 | ❌ 否 |
| **个人可用** | 不公开展示，但可进入作者自己的生成上下文 | ✅ 是 | ❌ 否 | ❌ 否 |
| **公开展示** | 可被他人阅读，但不一定进入生成池 | ❌ 否 | ❌ 否 | ✅ 是 |
| **公共生成池** | 可被平台用于公共世界生成（同时自动公开展示） | ❌ 否 | ✅ 是 | ✅ 是 |

## 七、知识库维护说明

### 入库流程（MVP 手动 / 后续自动化）

1. 用户上传文本（粘贴或上传文件）。
2. 系统自动提取：类型标签候选、文风标签候选、意象关键词。
3. 用户确认或编辑标签，选择权限状态。
4. 文本写入知识库，生成索引，供后续生成召回。

### 召回策略（技术提示）

- 根据用户 `idea` 中的关键词、情绪向量、类型倾向，从知识库中召回最相关的 3～5 条文本。
- 召回内容进入生成上下文，优先级高于通用模型知识。
- 如果召回内容中包含角色、地点、物件定义，系统应尽量保持一致性，避免矛盾。

### 去重与更新

- 相同正文重复上传时，保留最早版本，并可提示用户。
- 允许用户更新自己条目的权限或标签（建议记录修改历史）。
